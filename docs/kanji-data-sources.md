# Kanji Data Sources

This document describes the data sources used for Kanji stroke order, Jouyou classification,
and JLPT level classification in the Practice Sheet Builder app.

---

## 1. KanjiVG (Stroke Order SVG Data)

**Source:** https://github.com/KanjiVG/kanjivg

**License:** Creative Commons Attribution-ShareAlike 3.0 (CC BY-SA 3.0)
See the full license at https://creativecommons.org/licenses/by-sa/3.0/

**Usage in this app:**
- SVG files are stored locally in `/public/kanjivg/` as `{5-digit-hex}.svg`
- Each file contains stroke-path data for one Unicode character
- Stroke order is loaded on demand when a character is selected
- Stroke order is **never inferred from fonts** — it always comes from KanjiVG SVG data
- The KanjiVG manifest and index are generated by `scripts/generate-kanji-index.mjs`

**Coverage:**
- Hiragana: U+3041–U+309F
- Katakana: U+30A0–U+30FF
- Kanji: depends on locally available SVG files

**Attribution requirement:**
If distributing this app or derived works, you must credit KanjiVG per the CC BY-SA 3.0 license.

## 2. Jouyou Kanji (常用漢字)

**Source:** Japanese Agency for Cultural Affairs (文化庁)
Official table: https://www.bunka.go.jp/kokugo_nihongo/sisaku/joho/joho/kijun/naikaku/kanji/

**Stored in:** `src/data/source/joyoKanji.json`

**Notes:**
- The official Jouyou Kanji list contains 2,136 characters (revised 2010).
- The local source file currently contains only the subset of Jouyou Kanji that have
  matching KanjiVG SVG files in `/public/kanjivg/`. This is a local subset — not the full list.
- Grade 1–6 = Kyouiku Kanji (primary school); "S" = secondary-school Jouyou Kanji.
- To expand coverage: add more SVG files to `/public/kanjivg/` and expand `joyoKanji.json`,
  then run `npm run generate:kanji`.

---

## 3. JLPT Kanji Classification

**Source:** Curated reference list

**Stored in:** `src/data/source/jlptKanji.json`

**Important caveat:**
> JLPT does not publish an exhaustive official kanji list per level.
> The JLPT level classifications in this app are **reference classifications**
> based on commonly used study references and should **not** be treated as
> official JLPT certification data.

**Notes:**
- The local source file currently contains only the subset matching available KanjiVG SVG files.
- A kanji is assigned to at most one JLPT level; duplicates are reported by the generation script.
- To expand coverage: add more entries to `jlptKanji.json` and run `npm run generate:kanji`.

## 4. hanzi-writer-data-jp (Reference Only)

**Source:** https://github.com/chanind/hanzi-writer-data-jp

This dataset was reviewed as a potential data source. It is **not used** in this app's
stroke-order rendering pipeline. The app uses KanjiVG SVG files exclusively for stroke data.

If you add hanzi-writer-data-jp in the future:
- Check its license (MIT for code, CC BY-SA 4.0 for data derived from KanjiVG).
- Do not mix its path format with KanjiVG SVG paths in the same rendering pipeline.

> JLPT dataset unless an official source is added.
> Stroke order is always derived from KanjiVG — never from font glyphs.

## 6. Expanding the Kanji Library

To add more Kanji stroke-order data:

1. Download SVG files from the KanjiVG GitHub releases.
2. Place them in `/public/kanjivg/` using the `{5-digit-hex}.svg` naming convention.
3. Add Jouyou classification to `src/data/source/joyoKanji.json` if applicable.
4. Add JLPT classification to `src/data/source/jlptKanji.json` if applicable.
5. Run: `npm run generate:kanji`
6. Run: `npm run validate:kanji`
7. Commit the updated generated files.
